# Moranta UX Design Principles

## Purpose

This document defines the emotional and experiential principles that guide all Moranta user experiences.

The goal is to create technology that feels:

- calm
- intentional
- reflective
- emotionally safe
- empowering

rather than:

- addictive
- manipulative
- overwhelming

---

# Core UX Philosophy

Moranta is not a social media platform.

Moranta is not an attention platform.

Moranta is a reflective growth platform.

Every interaction should support awareness rather than consumption.

---

# Principle 1: Calm First

The interface should reduce cognitive load.

Users should never feel rushed.

Avoid:

- flashing animations
- aggressive notifications
- excessive movement
- visual clutter

Prefer:

- whitespace
- soft transitions
- clear hierarchy
- intentional pacing

---

# Principle 2: Reflection Before Interpretation

Users should engage with their own thoughts before receiving AI insights.

Examples:

Dream Journal:
- User reflects first.
- AI responds second.

Reflection Journal:
- User writes first.
- AI summarizes later.

---

# Principle 3: Human Sovereignty

Users remain in control.

Users decide:

- AI guidance level
- memory permissions
- personalization settings
- content visibility

The interface should reinforce autonomy.

---

# Principle 4: Emotional Safety

Interactions should feel:

- grounded
- compassionate
- respectful

Avoid:

- fear
- urgency
- shame
- guilt

---

# Principle 5: Meaningful Progress

Progress should focus on:

- consistency
- reflection
- awareness

Not:

- streak obsession
- dopamine loops
- competitive metrics

---

# Principle 6: Accessibility

Design for all users.

Prioritize:

- readability
- contrast
- keyboard navigation
- screen readers
- reduced motion options

---

# Principle 7: Human Wisdom Matters

AI should never dominate the experience.

Human voices, creators, and community should remain visible throughout the platform.
